The detection model —
classes & data

This is the longest pole in the build. A hazard the model never learned is a hazard it will never flag — so the model is only ever as good as the classes we define and the images we label. Here's the hazard vocabulary, the tiered rollout, and the data strategy that turns the glasses themselves into the moat.

3–5 weeks to a v1 model Recall-first Data is the moat

Recall is the metric that matters

In most vision products a false positive is an annoyance and a false negative is a shrug. Here a false negative is a hazard that walked past unflagged — a safety failure. So we optimize for recall on the safety-critical classes and hold a floor per class. But recall isn't free: a model that cries wolf gets muted, and a muted model catches nothing. The whole job is buying high recall without spending the worker's trust.

Why worksite vision is hard

ChallengeWhat it does to the model — and the answer
Small, distant objectsA hard hat 15 m away is a few pixels. Keep input resolution up, tile the frame, and weight small-object recall in training.
Occlusion & clutterPPE and guards get half-hidden by bodies and gear. Needs varied, real-world angles — exactly what stock data lacks.
POV motion blurA head-worn camera never holds still. Train on motion-blurred frames; don't only label crisp stills.
Class imbalanceHazards are rare in footage — thousands of "fine" frames per violation. Mine hard negatives; oversample the rare classes.
The domain gapPublic data is CCTV and stock photography; the product sees worker-POV. This is the core problem — and the opportunity below.

The insight worth building around

The glasses are the data flywheel

Every competitor can download the same public datasets — so a model trained only on them isn't a moat, it's a commodity. What no one else has is worker-POV footage of your customers' actual hazards, captured by the same camera that will run inference. Wire the pipeline so every pilot frame can be reviewed, labeled, and folded back into training. The product improves itself the more it's worn.

Labeling & evaluation

Labeling pipeline

  • 01Ontology first. Precise class definitions with the edge cases written down — partial PPE, distance thresholds, "when is a cable a trip hazard."
  • 02Tooling. CVAT / Roboflow / Labelbox for bounding boxes; a schema every annotator shares.
  • 03QA. Multi-annotator on a sample, measure inter-annotator agreement, adjudicate disagreements.
  • 04Active learning. The model surfaces its most uncertain frames; label those next, not random ones.
  • 05Volume. ~1–2k instances per class to start; more for the hard, imbalanced ones.

Evaluation that tells the truth

  • ✓Split by site, not at random. Same-site frames leak; a held-out worksite is the only honest generalization test.
  • ✓Headline metric: per-class recall at a fixed precision, with an agreed floor for critical classes (e.g. ≥ 0.90 hard-hat recall).
  • ✓False-negatives are the review. Every miss on a critical class gets looked at — that's the safety audit.
  • ✓On-device, not lab. Measure the quantized model at target FPS on the phone, not the full model on a workstation.

People are in every frame

Detecting PPE means detecting people. That carries obligations, and getting them right is also what makes the product deployable: process on-device, never run face recognition or identify individuals, keep only what training needs, and get site consent before any capture. Design this in from the first labeled frame — retrofitting privacy is how safety tools get banned from the floor.

What Phase 2 ships. A labeled dataset v1 (Tier 1 classes), a quantized Core ML / TFLite model clearing the agreed recall floor on a held-out site at target FPS, an evaluation report that leads with false-negatives, and a running capture → label → retrain loop. That model drops straight into the Phase 1 pipeline and hands Phase 3 something real to reason about. The dataset — not the model — is the asset that compounds.

Dataset references: SH17 (8,099 images / 75,994 instances / 17 classes, research-use license), Roboflow Universe Construction Site Safety, and CHV colored-hardhat/vest sets.
